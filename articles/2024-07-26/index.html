Go 1.23リリース連載 range over funcとiterパッケージ | フューチャー技術ブログ

💡 Go 1.23 の新機能を紹介します。 ### range over func Go 1.23 では、関数から値を生成するために、range を使用できるようになりました。これにより、関数を繰り返し処理したり、イテレータのように扱ったりできます。 ```go func someFunc() func() (int, bool) { i := 0 return func() (int, bool) { if i >= 10 { return 0, false } v := i i++ return v, true } } for v, ok := range someFunc(); ok; v, ok = someFunc() { fmt.Println(v) } ``` ### iter iter は、イテレーターを簡単に生成できるようにするために追加されたパッケージです。iter パッケージには、シーケンス (sequence) という概念が導入されています。 シーケンスは、イテレーターを使用して値を生成する関数のことです。シーケンスは、range over func を使用して繰り返し処理できます。 ```go package iter type Seq[T any] interface { ~func(yield func(T) bool) } // 以下に記載されているように関数を定義することで、 // range over func の利用方法が可能となる。 // func someFunc() Seq[int] { /* ... */ } ``` また、iter パッケージには、range over func と組み合わせて使用できる型定義も提供されています。 ```go package iter type Seq2[K any, V any] interface { ~func(yield func(K, V) bool) } // 以下に記載されているように関数を定義することで、 // range over func の利用方法が可能となる。 // func someFunc() Seq2[int, int64] { /* ... */ } ``` ### Proposal Go 1.23 から導入された range over func と iter パッケージについて、Proposal から利用例までを紹介します。 * Proposal では、range over func の導入によって Go の反復処理が統合されると述べられています。 * Go1.23 以降は、slices と maps に対する range over func の使用が推奨されるとのことです。
